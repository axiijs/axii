# Error Handling

当前项目是一个响应式前端框架，它的特点：
1. 和渲染框架无关的响应式的数据结构。
2. 依托于响应式式数据结构实现的精确更新。

**产出的任何默认输出到 prompt/output 下**

## Task 1 error 方案设计
现在发现，一点业务逻辑到页面上渲染更新时出错了，经常会报：
```
can not find nextSibling
```
我们很难知道是哪个组件哪段逻辑写错了。
你来完全掌握整个框架，提出一个更好的服务于开发者的报错方案。注意：一定要结合现代最流行的技术。
把你的方案写成一个独立的文档。
